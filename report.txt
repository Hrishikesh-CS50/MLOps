# MLOps Repository Setup and Version Control Workflow Report

## 1. Project Initialization and Environment Setup

The project began with the creation of a local repository named **MLOps**. A Python virtual environment was created and activated to ensure dependency isolation and reproducibility.

A Jupyter Notebook file, `sample1.ipynb`, was created and populated with initial code. Upon execution, the notebook indicated that certain required libraries were not installed. These dependencies were installed using `pip`, after which the code executed successfully.

To ensure consistent dependency management, a `requirements.txt` file was generated listing all required libraries.

---

## 2. Git Repository Configuration and Initial Commit

The repository was initialized using Git. The following files were staged and committed:

* `sample1.ipynb`
* `requirements.txt`

A remote repository was then created on GitHub, and the local repository was connected to the remote origin. The default branch was set to `main`, and the initial commit was pushed successfully.

---

## 3. Branching Strategy and Pull Request Workflow

A feature branch was created and checked out to follow best practices for collaborative development.

Within the feature branch:

* A `README.md` file was created, added, committed, and pushed.
* A `.gitignore` file was created to exclude the virtual environment (`venv`) directory from version control.
* The `.gitignore` file was staged and committed.

After pushing the feature branch to GitHub, a Pull Request (PR) was created to merge the changes into the `main` branch.
